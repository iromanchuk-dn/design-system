# Welcome to the DAP Design System

Welcome to our design system documentation! This guide will help you understand how to use our component library effectively, whether you're a developer implementing components or a designer creating new interfaces.

## For Developers

### Getting Started

1. **Using the Design System**

**Within the Monorepo**

```tsx
// No installation needed! Just import directly:
import { DsButton, DsIcon } from '@drivenets/design-system';
```

**Outside the Monorepo**

```bash
# The library is available as an NPM package
npm install @drivenets/design-system
# or
yarn add @drivenets/design-system
# or
pnpm add @drivenets/design-system
```

2. **Example Usage**

```tsx
import { DsButton, DsIcon } from '@drivenets/design-system';

function MyComponent() {
	return (
		<DsButton>
			<DsIcon name="check" />
			Click me
		</DsButton>
	);
}
```

3. **Setting Up Styles**

   To use the design system styles in your project, you need to configure your build tool to import them globally. Here's how to set it up:

   **Vite Configuration**

   ```ts
   // vite.config.ts
   export default defineConfig({
   	css: {
   		preprocessorOptions: {
   			scss: {
   				loadPaths: ['libs/web/src/styles'],
   				additionalData: '@use "@drivenets/design-system/styles/styles.scss" as *;',
   			},
   		},
   	},
   });
   ```

   **Webpack Configuration**

   ```js
   // webpack.config.js
   module.exports = {
   	module: {
   		rules: [
   			{
   				test: /\.scss$/,
   				use: [
   					'style-loader',
   					'css-loader',
   					{
   						loader: 'sass-loader',
   						options: {
   							loadPaths: ['libs/web/src/styles'],
   							additionalData: '@use "@drivenets/design-system/styles/styles.scss" as *;',
   						},
   					},
   				],
   			},
   		],
   	},
   };
   ```

   This setup ensures that:
   - Design system styles are available globally
   - Variables, mixins, and functions are accessible
   - Consistent theming across your application
   - Uses NX's path resolution for better maintainability

### Development Workflow

1. **Creating New Components**

```bash
npx nx generate @dap-workspace/dap-plugin:ds-component my-component
```

2. **Running Tests**

```bash
nx test web
```

3. **Storybook Development**

```bash
nx run web:storybook
```

### Best Practices

- Always use the provided components instead of creating custom ones
- Follow the component API documentation for proper usage
- Use TypeScript for better type safety and developer experience
- Write tests for new components
- Document components with stories and examples
